2.4. The Edit Tab

2.4.1 The Edit Tab View

The Edit tab is where the user will spend the bulk of their time. Available to admins only, the tab's primary function is to allow for rearranging and rescheduling of Sections in the Schedule.

The bulk of the tab's area is occupied by a grid that displays the Schedule. The user can pan around the grid by pressing the space bar and dragging, middle-clicking and dragging, or by using the scroll bars.

Each Section is represented by a small colored square. Each square also has a Pin button on it, explained further along in this document. As the user moves their mouse over the grid, the time block they are currently hovering over, whether it contains a Section or is blank, is highlighted with a bold rectangle.

The horizontal axis is always time. As the user moves their mouse over the grid, the time block the cursor is currently on is displayed on the horizontal axis.

The vertical axis represents Rooms by default, but the user can change this to Courses or Instructors by clicking on a series of tabs arranged vertically down the left-hand side.

2.4.2 Adding, Editing and Deleting Sections

To edit a section, the user double-clicks the section's square. To create a section, the user double-clicks a blank space on the Schedule. Both these actions bring the user to the Edit Section screen. When creating a new section, the fields in the Edit Section screen will be blank; when editing an existing section, the fields will be populated with the current Section properties.

[Image: the Edit Section screen. It occupies the entire Edit Tab, and has fields for the four Section properties. These fields are currently blank]

Here, the user can change all aspects of a Section: the Time, Instructor, Course and Room.

[Image: the Edit Section screen. Now all fields are filled in with data]

To save changes, the user clicks the Save button. To cancel changes without saving, the user clicks the Cancel button.

To delete the Section, the user clicks the Delete This Section button. After confirming the deletion, the Section will be deleted.

Once Save, Cancel or Delete is selected, the user will be returned to the main Edit Tab view.

2.4.3 Refining a Schedule: Pin & Generate

The Pin & Generate loop is an integral part of the KCScheduler tool. Because schedule that is generated by software will rarely be ideal, the schedule must be refined by the user. The Pin & Generate method offered by the KCScheduler tool is an intuitive way to iteratively improve the Schedule.

First, the user Generates a Schedule (see 2.2.5 Generate Schedule). This will take the user to the Edit tab, where they will probably discover some inadequacies in the generated the Schedule. Hopefully, however, some elements of the Schedule will be suitable. The user will then Pin the sections they find acceptable. Pinned Sections will not be touched by the Schedule Generator.

Then, the user will run the Schedule Generator again. New constraints may be put into place on this point. However, even if the constraints are not changed, the Schedule Generator will scramble the un-Pinned sections, yielding another Schedule which, hopefully, is marginally better than the previous one.

The user can then continue to refine and edit until arriving at a suitable Schedule.
