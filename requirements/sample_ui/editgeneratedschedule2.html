2.x Auto-Scheduling


The auto-scheduler uses a course list, a professor list, and a room list to create a schedule. The auto-scheduler will generate a schedule that fits all the requirements given by the user, but this schedule will usually need to be adjusted for conflicts, additional subjective factors, etc.


2.x.1 Running the Auto-Scheduler


To run the auto-scheduler, click the Auto-Scheduler button. This will bring up the following screen:


(nyerr)


Here's how to adjust each parameter, blah blah blah. You pick which lists of classrooms, teachers, and courses it will work with. There are buttons that link to the list-editing screens for convenience.


Once all parameters are set, the user clicks the Run Auto-Scheduler button. The scheduler will create a new schedule and display it for editing.


2.x.2 Pinning Sections and Re-Generating


You can edit a generated schedule manually to your satisfaction, but often it's best to take an iterative approach to auto-scheduling.


During auto-scheduling, each section has a pin button on it. Pinning a section will tell the auto-scheduler that you like the placement of that section and that it should remain untouched by the scheduler.


(Figure four million: the pin button. It looks like a thumbtack on the section)


Pin all the sections you're satisfied with, then click the Run Auto-Scheduler button again. The auto-scheduler will scramble all unpinned sections. Now, pin any that are satisfactory, then run the auto-scheduler again. Continue in this way until the schedule is roughed out and ready to be edited by hand.
