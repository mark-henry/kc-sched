1.1. Problem Statement

Currently Cal Poly University has over 50 departments, and within each department is at least one worker who must schedule courses to be offered each quarter. This task is done 50 different ways, varying from expensive software to bare-bones pen and paper. Our group sees the following issues that we’d like to resolve:

a.       Inconsistent tools and methodology.

b.      Time intensive data entry processes.

While the departments could continue with the same methodology, and the process would continue to function, many of the people we interviewed felt that if this task was no longer their problem, they wouldn’t be disappointed.  Their main complaints seemed to be:

a.       No current methods allow them to easily export to the “PeopleSoft” forms they must enter section information into.

b.      Figuring out what classes to offer when in a day takes a long time, and there are optimal ways to schedule classes and their prerequisites.

Our scheduling tool aims to fix the problems mentioned above, and address the wishes of the users within each department. This project should result in a tool that is easy to use, saves users time, and leaves students with optimal course offerings each quarter.
