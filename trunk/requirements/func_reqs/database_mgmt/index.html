2.2.2. The Data Tab

There are three major databases in our application. One database consists of all instructors, one consists of all rooms, one consists of all courses. These databases need to be maintained by an administrator (i.e. the department scheduler). This individual or individuals will be responsible for adding the appropriate items to each database initially. This is an initial start-up cost. All rooms, instructors and courses will need to be manually entered into these databases. Although this is extremely tedious, it is a one-time task. Once all items have been added to the databases, they can be viewed, edited, added to, deleted from, etc. These UI scenarios display those types of functionality within our application. It is important to note that editing of this information is a copy of the permanent information stored in the central database. This information can change per schedule and thus is a copy. The administrator does have access to edit all permanent information. The instructors only have access to the permanent versions of their personal preferences.
