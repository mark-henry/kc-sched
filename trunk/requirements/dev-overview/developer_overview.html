4. Developer Overview


Major Objects
Schedule - Container of many sections which organizes all of them on a grid layout.
Section - Container of information to be placed in a schedule.  Can be edited individually or moved as a whole object.
Instructor - Information on an instructor, including: name, phone number,  office and preferences.
Course - Information on a course, including: name, number, department and units.
Room - Information on a room, including: building and room number, capacity, lab stations and key features in the room.

Major Operations
Adding / Removing Instructor, Course and Room Information
Magic Generate Button
Editing of generated schedule
Exporting a generated schedule

The hardest part to implement should be the editing of a generated schedule.  To allow the user a drag and drop functionality, the ability to pin some sections so they can not be moved and the ability to refactor the schedule around those pinned sections will all require a great deal of work.  Also, throughout the whole process the validity check needs to run so that any change a user makes is checked against the rest of the schedule to see if that change can work.

Structure
Each schedule is composed of many sections.  Each section is in turn composed of one instructor, one course, one room, and a set of time information.  All of the information contained within a section is editable from the schedule and doing so will only edit the preferences or information saved to that schedule.  If a user wants to edit the permanent data they can do so in the data tab.  This allows for easy distinction between permanent and temporary data management.  An administrative user is unable to change the permanent preferences for instructors in the database.
