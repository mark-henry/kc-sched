2.2.6 Edit Tab

2.2.6.1 Edit Tab Overview

The Edit tab is where the user will spend the bulk of their time. Available to admins only, the tab's primary function is to allow for rearranging and rescheduling of Sections in the Schedule.

The bulk of the tab's area is occupied by a grid that displays the Schedule. Each Section is represented by a small colored square. Each square also has a Pin button on it, explained further along in this document.

[Image: general overview of the Edit tab]

The horizontal axis is always time. The vertical axis represents Rooms by default, but the user can change this to Courses or Instructors by clicking on a series of tabs arranged vertically down the left-hand side.

[Two images demonstrating the Course and Instructor views]

The Error list pops out of the right-hand side of the tab if there are Errors to display; otherwise, it is hidden.

To change the scheduling of a Section, the user simply clicks and drags the Section into a new time slot. To edit the Section, the user can double-click the Section.

Information for a section is displayed when the cursor is on a Section. The Course, Instructor, and Room is displayed in this pop-up.

2.2.6.2 Refining a Schedule: Pin & Generate

The Pin & Generate loop is an integral part of the KCSched tool. Because schedule that is generated by software will rarely be ideal, the schedule must be refined by the user. The Pin & Generate method offered by the KCSched tool is an intuitive way to iteratively improve the Schedule.

First, the user Generates a Schedule (See 2.2.5 Generate Schedule). This will take the user to the Edit tab, where they will discover the inadequacies of the Schedule. However, some elements of the Schedule will be suitable. The user will then Pin the sections they find acceptable. Pinned Sections will not be touched by the Schedule Generator.

Then, the user will run the Schedule Generator again. New constraints may be put into place on this point. However, even if the constraints are not changed, the Schedule Generator will scramble the un-Pinned sections, yielding another Schedule which, hopefully, is marginally better than the previous one.

The user can then continue to refine and edit until arriving at a suitable Schedule.
